# Changelog

本文件记录 VSSWeb 各版本面向用户的主要变化。

格式参考 [Keep a Changelog](https://keepachangelog.com/zh-CN/1.1.0/)，版本号遵循发布标签命名。

## [v0.2] - 2026-04-29

### Added

- 增加设备网络预检查，在尝试登录前先检测目标地址是否可达，减少无响应或直接失败时的排查成本。
- 增加常用 Web 访问快捷入口，便于从客户端快速打开设备相关页面。
- 增加可配置日志控制，可按需要调整日志输出行为，降低日常使用时的噪音。

### Changed

- 更新提示与主窗口标题的版本展示方式更加明确，便于识别当前运行版本和可用更新版本。

### Fixed

- 修复 ActiveX 启动流程中的稳定性问题，降低控件初始化失败或启动顺序异常的概率。
- 修复嵌入式 Web 宿主与 ActiveX 承载之间的隔离问题，减少页面与控件互相影响导致的异常行为。

### Technical

- 新增 `lib/netcheck.aardio`，将设备连通性预检查接入主流程。
- 调整日志配置与日志模块实现，使日志开关和输出控制集中到配置层管理。
- 重构嵌入式 Web 承载相关代码，梳理 `main.aardio`、`dlg/web.aardio` 与 `dlg/native.aardio` 的分工边界。

## [v0.1] - 2026-04-28

### Added

- 提供 Windows 桌面版监控客户端，用于承载 `IVSWeb` ActiveX 监控控件的核心能力。
- 支持通过连接地址、用户名、密码登录设备。
- 支持启动后在配置完整时自动连接设备并打开全部通道。
- 支持一键打开全部通道、一键关闭全部通道。
- 支持 1 至 8 路通道的单独打开与关闭，按钮状态会随通道状态变化更新。
- 支持从设备读取并显示通道名称，提升多通道识别体验。
- 支持云台方向控制，包括上、下、左、右方向操作。
- 支持打开常用设备功能入口，包括系统设置、录像、警报和关于窗口。
- 提供连接设置窗口，可保存设备连接地址、用户名和密码。
- 设置窗口密码框支持临时显示和隐藏，便于核对输入内容。
- 设置窗口点击“取消”时不会断开设备或重新登录。
- 设置窗口点击“保存”但连接地址、用户名、密码均未变化时，不会刷新主界面或重新登录。
- 只有连接配置实际变化并保存后，才会重新加载配置并重新连接设备。
- 密码隐藏样式在显示/隐藏切换后保持与初始样式一致。
- 主窗口标题显示当前发布版本号，便于用户确认正在运行的版本。
- 提供内置日志区域，用于查看连接、调用和状态变化信息。
- 支持在线更新检查，在检测到新版本准备就绪时提示用户。

### Technical

- 使用 `web.form` 承载极简 HTML 页面，在 IE 内核中创建 `IVSWeb` ActiveX 控件。
- 通过 `<object>` 初始化阶段传入 ActiveX 参数，保留 `lVideoWindNum=4` 以维持默认四分屏初始化行为。
- 通过 `wb.doScript(...)` 从 aardio 调用页面内 JavaScript，再调用 ActiveX 方法。
- 通过 `window.external` 接收 ActiveX 事件回调，并同步通道按钮状态。
- 将配置、日志和通道名称处理拆分到 `lib/` 模块，降低主程序复杂度。
- 保留 `dlg/native.aardio` 作为纯 `createEmbed()` 对照实现，保留 `dlg/exportOCX.aardio` 用于重新导出 ActiveX 类型信息。
- 增加发布流程文档与 GitHub Actions 自动发布流程，推送版本 tag 后可自动创建 GitHub Release。

### Notes

- 运行前需要本机已安装并注册 `CLSID:D639FA00-CB11-4f67-82F2-C0A87EAECD13` 对应的 `IVSWeb` ActiveX 控件。
- aardio 程序位数需要与 ActiveX 控件位数一致。
- 当前版本没有自动化测试，发布前以人工验证设备连接、通道预览、云台控制和设置保存为准。
